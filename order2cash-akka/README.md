## What this project is

Order-to-Cash workflow implemented as five Akka actors (Buyer, Supplier, Inventory, Logistics, Bank) communicating over **Akka Remoting (Artery/TCP)**. Each actor runs in its own Docker container. XML documents flow between actors; each step applies an XSLT transformation to produce the next document type. Every document is persisted to `data/output/<flowId>/` on a shared bind-mount.

## Commands

### Build the fat JAR
```bash
mvn package -DskipTests
```

### Run all containers
```bash
docker-compose up --build
```

### Trigger a workflow
Wait for all five containers to log "Started.", then copy any PurchaseOrder `.xml` into `data/input/`:
```bash
cp data/input/sample-po.xml data/input/po-$(date +%s).xml
```
The Buyer detects the file, assigns a flow GUID, and kicks off the chain. All eight documents appear under `data/output/<flowId>/` as the flow progresses.

### Watch the flow
```bash
docker-compose logs -f buyer supplier inventory logistics bank
```

All five containers mount `./data:/app/data` so they share the same filesystem tree.

### Flow ID
`UUID.randomUUID()` generated by `BuyerActor` when a new PurchaseOrder file is detected. Carried in every Akka message as `msg.flowId` and used as the output subdirectory name by `DocumentWriter.write(flowId, docName, xml)`.

### Akka Remoting
All containers share ActorSystem name **`order2cash`** and listen on port `2551`. Remote paths:
```
akka://order2cash@<hostname>:2551/user/<actorName>
```
`RemoteAddresses.java` builds these from `*_HOST` env vars (defaults match docker-compose service names).

### Key files
- `com.order2cash.Main` вҖ” reads `AGENT_TYPE`, creates `ActorSystem("order2cash")`, instantiates the matching actor.
- `com.order2cash.remote.RemoteAddresses` вҖ” builds `ActorSelection` remote paths.
- `com.order2cash.util.XsltTransformer` вҖ” loads XSLT from classpath `/xslt/`, applies `javax.xml.transform`.
- `com.order2cash.util.DocumentWriter` вҖ” writes XML to `data/output/<flowId>/`; base path overridable via `OUTPUT_DIR`.

## Environment variables
| Variable | Default | Purpose |
|----------|---------|---------|
| `AGENT_TYPE` | `buyer` | Actor to instantiate: `buyer`, `supplier`, `inventory`, `logistics`, `bank` |
| `AKKA_HOSTNAME` | `127.0.0.1` | Hostname Akka Artery binds and advertises |
| `AKKA_PORT` | `2551` | Artery TCP port |
| `PURCHASE_ORDER_DIR` | `data/input` | Directory Buyer watches for `.xml` files |
| `OUTPUT_DIR` | `data/output` | Base directory for per-flow output subfolders |
| `BUYER_HOST` | `buyer` | Hostname of the Buyer container |
| `SUPPLIER_HOST` | `supplier` | Hostname of the Supplier container |
| `INVENTORY_HOST` | `inventory` | Hostname of the Inventory container |
| `LOGISTICS_HOST` | `logistics` | Hostname of the Logistics container |
| `BANK_HOST` | `bank` | Hostname of the Bank container |
